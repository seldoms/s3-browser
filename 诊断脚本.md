# S3 连接诊断脚本

## 🔍 问题: 能连上但读不到桶列表

这个脚本会帮助您诊断问题。

## 📋 诊断步骤

### 步骤 1: 查看原始 API 响应

**打开这个页面:**
```
http://localhost:3001/raw-data.html
```

**操作:**
1. 点击 "📡 获取原始 API 响应"
2. 查看右侧的"响应数据"
3. 查看"数据分析"部分
4. 查看"诊断建议"

**重点关注:**
- `buckets` 字段是否存在?
- `buckets` 的值是什么? (null / undefined / [] / 数组)
- 是否有错误信息?

### 步骤 2: 查看服务器日志

**在终端中查看:**

服务器日志会显示:
```
[API] 开始执行 listBuckets
[API] S3 原始响应: { ... }
[API] response.Buckets: [...]
[API] Buckets 类型: object
[API] Buckets 是否为数组: true
[API] Buckets 长度: X
[API] 返回结果: { buckets: [...] }
```

**如果看到 "Buckets 长度: 0":**
- ✅ API 工作正常
- ⚠️ 该账户下确实没有桶
- 💡 需要创建桶或检查 Region

**如果看到 "Buckets: undefined" 或 "Buckets: null":**
- ⚠️ S3 API 返回了空值
- 💡 可能是权限问题或 API 实现问题

### 步骤 3: 查看浏览器控制台

**打开 Explorer 页面:**
```
http://localhost:3001/explorer
```

**按 F12 打开控制台,查找:**
```
[fetchBuckets] 开始获取桶列表
[fetchBuckets] API 返回数据: { buckets: [...] }
[fetchBuckets] 桶数量: X
[useEffect] buckets 状态已更新, 数量: X
```

**如果 "桶数量: 0":**
- ✅ 前端正常接收到数据
- ⚠️ 数据确实是空的
- 💡 问题在于后端返回空数据

**如果没有看到这些日志:**
- ⚠️ fetchBuckets 没有被调用
- 💡 可能是 React 组件问题

## 🎯 常见情况分析

### 情况 A: buckets 是空数组 []

**症状:**
- API 返回 `{ buckets: [] }`
- 桶数量显示 0
- 没有错误信息

**原因:**
1. **该账户下确实没有桶** (最常见)
2. **Region 配置错误** - 桶在其他区域
3. **凭证权限不足** - 只能访问特定桶

**解决:**
```
方案 1: 创建测试桶
  → 在 Explorer 点击 "创建第一个存储桶"
  
方案 2: 检查 Region
  → 打开 debug.html
  → 运行完整诊断
  → 对比云服务商控制台的 Region
  
方案 3: 测试已知桶
  → 打开 debug.html
  → 使用"手动测试桶访问"
  → 输入已知的桶名称
```

### 情况 B: buckets 是 null 或 undefined

**症状:**
- API 返回 `{ buckets: null }` 或 `{ buckets: undefined }`
- 控制台显示 "Buckets: null"

**原因:**
1. **S3 SDK 返回异常**
2. **凭证没有 ListBuckets 权限**
3. **Endpoint 配置问题**

**解决:**
```
1. 检查凭证权限
   → 登录云服务商控制台
   → 查看 Access Key 的权限策略
   → 确保有 s3:ListAllMyBuckets 权限

2. 检查 Endpoint
   → 确认 Endpoint 格式正确
   → 确认 Region 与 Endpoint 匹配

3. 查看服务器错误日志
   → 可能有异常被捕获但没有正确处理
```

### 情况 C: 有 buckets 但前端不显示

**症状:**
- raw-data.html 显示有桶
- Explorer 页面不显示

**原因:**
1. **前端渲染问题**
2. **状态更新问题**
3. **CSS 显示问题**

**解决:**
```
1. 检查浏览器控制台
   → 查看 [useEffect] buckets 状态已更新
   → 确认 buckets.length > 0

2. 检查 React DevTools
   → 查看组件的 buckets state
   → 确认数据已经设置

3. 强制刷新
   → Ctrl+Shift+R (Windows)
   → Cmd+Shift+R (Mac)
```

## 🔧 快速测试命令

### 测试 1: 直接 API 调用

在浏览器控制台执行:
```javascript
fetch('/api/s3', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    credentials: JSON.parse(localStorage.getItem('s3_creds')),
    action: 'listBuckets',
    params: {}
  })
})
.then(r => r.json())
.then(data => {
  console.log('API 响应:', data);
  console.log('buckets:', data.buckets);
  console.log('buckets 类型:', typeof data.buckets);
  console.log('是数组:', Array.isArray(data.buckets));
  console.log('长度:', data.buckets ? data.buckets.length : 'N/A');
});
```

### 测试 2: 检查 localStorage

```javascript
const creds = JSON.parse(localStorage.getItem('s3_creds'));
console.log('凭证:', creds);
console.log('Endpoint:', creds.endpoint);
console.log('Region:', creds.region);
```
