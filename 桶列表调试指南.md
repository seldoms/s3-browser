# 桶列表加载问题 - 调试指南

## 🔍 问题描述

登录成功后,explorer 页面没有显示存储桶列表。

## 🛠️ 调试步骤

### 方法 1: 使用调试页面

1. 打开浏览器访问: **http://localhost:3001/debug.html**
2. 页面会自动显示当前保存的凭证信息
3. 点击"测试获取桶列表"按钮
4. 查看返回的结果:
   - 如果显示"成功获取桶列表",说明 API 工作正常
   - 如果显示错误信息,说明凭证或配置有问题

### 方法 2: 查看浏览器控制台

1. 打开 **http://localhost:3001/explorer**
2. 按 `F12` 或 `Cmd+Option+I` (Mac) 打开开发者工具
3. 切换到 **Console** 标签页
4. 查找以下调试日志:

```
[fetchBuckets] 开始获取桶列表, creds: 存在
[fetchBuckets] 调用 callS3Api...
[callS3Api] 调用 action: listBuckets
[callS3Api] Web API 返回: { buckets: [...] }
[fetchBuckets] API 返回数据: { buckets: [...] }
[fetchBuckets] 桶列表: [...]
[fetchBuckets] 桶数量: X
[fetchBuckets] 已设置 buckets 状态
[useEffect] buckets 状态已更新, 数量: X
[useEffect] buckets 内容: [...]
```

5. 如果看到这些日志,说明数据获取成功
6. 如果桶数量为 0,可能的原因:
   - 该账户下确实没有存储桶
   - Region 配置错误
   - 凭证权限不足

### 方法 3: 检查 Network 请求

1. 在开发者工具中切换到 **Network** 标签页
2. 刷新页面
3. 查找 `/api/s3` 的 POST 请求
4. 点击该请求,查看:
   - **Headers**: 确认请求方法和 URL
   - **Payload**: 查看发送的凭证和参数
   - **Response**: 查看返回的数据

## 🔧 常见问题和解决方案

### 问题 1: 返回空的桶列表

**可能原因:**
- 该账户下确实没有创建任何存储桶
- Region 配置错误,无法访问该区域的桶

**解决方案:**
1. 登录云服务商控制台,确认是否有存储桶
2. 检查 Region 配置是否正确
3. 尝试在应用中创建一个测试桶(菜单: 存储桶 → 新建存储桶)

### 问题 2: API 返回错误

**可能原因:**
- Endpoint 配置错误
- Access Key 或 Secret Key 无效
- 网络连接问题
- Region 与 Endpoint 不匹配

**解决方案:**
1. 使用调试页面的"测试连接"功能
2. 检查凭证是否正确
3. 确认 Endpoint 和 Region 是否匹配
4. 查看 `连接问题排查.md` 文档

### 问题 3: 页面一直显示"加载中"

**可能原因:**
- API 请求超时
- 网络连接问题
- 服务端错误

**解决方案:**
1. 查看浏览器控制台是否有错误信息
2. 检查 Network 标签页中的请求状态
3. 尝试刷新页面

## 📝 收集诊断信息

如果问题仍然存在,请收集以下信息:

1. **浏览器控制台日志** (完整的 Console 输出)
2. **Network 请求详情** (特别是 `/api/s3` 的请求和响应)
3. **凭证配置信息** (不要包含 Secret Key!)
   - Endpoint
   - Region
   - Access Key ID (前10位)
4. **云服务商** (华为云/AWS/MinIO/其他)
5. **错误截图**

## 💡 提示

- 如果调试页面能成功获取桶列表,但 explorer 页面不显示,可能是前端渲染问题
- 如果两个页面都无法获取桶列表,则是 API 或凭证问题
- 空状态提示"未找到存储桶"表示 API 调用成功但返回了空列表
