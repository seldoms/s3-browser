# 未找到存储桶 - 解决方案

## 📋 问题说明

您已经成功登录,但系统提示"未找到存储桶"。这说明:

✅ **连接成功** - API 调用正常  
✅ **凭证有效** - 身份验证通过  
⚠️ **桶列表为空** - 该账户下没有存储桶

## 🎯 解决方案

### 方案 1: 创建新的存储桶 (推荐)

如果您是第一次使用,或者确实需要创建新桶:

1. 在 Explorer 页面左侧,点击 **"➕ 创建第一个存储桶"** 按钮
2. 或者使用菜单: **存储桶 → 新建存储桶**
3. 输入桶名称(只能包含小写字母、数字和连字符)
4. 点击创建

**桶命名规则:**
- 只能使用小写字母、数字、连字符(-)
- 长度 3-63 个字符
- 不能以连字符开头或结尾
- 示例: `my-test-bucket`, `data-2024`, `backup-files`

### 方案 2: 检查 Region 配置

如果您在云服务商控制台看到有桶,但应用中看不到:

**可能原因:** Region 配置错误,桶在其他区域

**解决步骤:**

1. 打开 **http://localhost:3001/debug.html** (调试工具)
2. 查看当前的 Region 配置
3. 登录云服务商控制台,查看桶所在的 Region
4. 如果不匹配,返回登录页重新配置正确的 Region

**常见 Region 对照:**

华为云 OBS:
- 北京四: `cn-north-4` → Endpoint: `https://obs.cn-north-4.myhuaweicloud.com`
- 上海一: `cn-east-3` → Endpoint: `https://obs.cn-east-3.myhuaweicloud.com`
- 广州: `cn-south-1` → Endpoint: `https://obs.cn-south-1.myhuaweicloud.com`

AWS S3:
- 美国东部: `us-east-1` → Endpoint: `https://s3.amazonaws.com`
- 美国西部: `us-west-2` → Endpoint: `https://s3.us-west-2.amazonaws.com`

### 方案 3: 手动测试已知桶

如果您知道桶的名称,可以直接测试访问:

1. 打开 **http://localhost:3001/debug.html**
2. 在"手动测试桶访问"区域输入桶名称
3. 点击"测试访问此桶"
4. 如果能访问,说明桶存在但 Region 配置可能有问题

### 方案 4: 检查凭证权限

**可能原因:** 凭证只有特定桶的权限,没有 ListBuckets 权限

**解决步骤:**

1. 登录云服务商控制台
2. 检查 Access Key 的权限策略
3. 确保有以下权限:
   - `s3:ListAllMyBuckets` (列出所有桶)
   - `s3:ListBucket` (列出桶内容)
   - `s3:GetObject` (读取对象)
   - `s3:PutObject` (上传对象)

4. 根据失败项进行修复

## 📝 常见场景

### 场景 1: 全新账户

**情况:** 刚注册的云服务账户,从未创建过桶

**解决:** 在应用中创建第一个桶即可

### 场景 2: 测试账户

**情况:** 使用的是测试凭证,没有实际的桶

**解决:** 
1. 创建测试桶
2. 或者更换为有桶的正式凭证

### 场景 3: 多区域部署

**情况:** 桶分布在不同的 Region

**解决:**
1. 记录每个桶所在的 Region
2. 切换 Region 后重新登录
3. 或者为每个 Region 保存不同的凭证配置

### 场景 4: 权限受限

**情况:** 使用的是受限凭证,只能访问特定桶

**解决:**
1. 使用调试工具的"手动测试桶访问"功能
2. 输入已知的桶名称直接访问
3. 或者联系管理员获取完整权限

## ✅ 验证步骤

完成上述操作后,请验证:

1. **刷新页面** - 按 F5 或点击刷新按钮
2. **查看左侧栏** - 应该能看到桶列表
3. **点击桶名** - 应该能看到桶内文件
4. **测试上传** - 尝试上传一个测试文件

## 🆘 仍然有问题?

如果尝试了所有方案仍然无法解决:

1. **收集信息:**
   - 云服务商名称(华为云/AWS/MinIO/其他)
   - Region 配置
   - 调试工具的诊断报告截图
   - 浏览器控制台的错误信息

2. **检查文档:**
   - `使用指南.md` - 基础配置说明
   - `连接问题排查.md` - 连接问题详解
   - `桶列表调试指南.md` - 调试方法

3. **常见错误信息:**
   - "NoSuchBucket" → 桶不存在或 Region 错误
   - "AccessDenied" → 权限不足
   - "InvalidAccessKeyId" → Access Key 无效
   - "SignatureDoesNotMatch" → Secret Key 错误

## 💡 最佳实践

1. **首次使用:** 先创建一个测试桶,确保一切正常
2. **多账户:** 为不同的云服务商保存不同的配置
3. **定期检查:** 确保凭证没有过期
4. **备份配置:** 记录 Endpoint 和 Region 配置
